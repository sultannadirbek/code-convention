Стандарты Документации

Эта страница описывает стандарты для генерации документации с использованием l5-swagger в Laravel. L5 Swagger автоматически создаёт документацию на основе аннотаций в коде контроллеров. Ниже приводится пример полного описания эндпоинта для создания нового заказа.

1. Основные элементы l5-swagger-документации

Для корректного написания документации необходимо описать следующие элементы в аннотациях:

  • Summary: Краткое описание цели эндпоинта.
  • Description: Подробное описание работы эндпоинта.
  • Method: HTTP-метод (POST, GET и т.д.).
  • Parameters: Входные параметры в теле запроса, path или query параметры.
  • Responses: Возможные ответы от API, включая коды HTTP (например, 200, 400, 404).
  • Tags: Категории для группировки эндпоинтов (например, "Заказы").
  • RequestBody: Описание схемы данных для POST или PUT запросов.

2. Пример документации для создания заказа (Laravel)

Пример аннотаций для метода создания нового заказа в контроллере Laravel:


/**
 * @OA\Post(
 *     path="/api/orders",
 *     summary="Создать новый заказ",
 *     description="Эндпоинт для создания нового заказа на основе данных пользователя и товаров.",
 *     tags={"Заказы"},
 *     @OA\RequestBody(
 *         required=true,
 *         description="Данные для создания нового заказа",
 *         @OA\JsonContent(
 *             type="object",
 *             required={"userId", "items", "shippingAddress"},
 *             @OA\Property(
 *                 property="userId",
 *                 type="integer",
 *                 description="ID пользователя, создающего заказ",
 *                 example=1
 *             ),
 *             @OA\Property(
 *                 property="items",
 *                 type="array",
 *                 description="Список товаров",
 *                 @OA\Items(
 *                     type="object",
 *                     @OA\Property(
 *                         property="productId",
 *                         type="integer",
 *                         description="ID товара",
 *                         example=101
 *                     ),
 *                     @OA\Property(
 *                         property="quantity",
 *                         type="integer",
 *                         description="Количество товара",
 *                         example=2
 *                     )
 *                 )
 *             ),
 *             @OA\Property(
 *                 property="shippingAddress",
 *                 type="string",
 *                 description="Адрес доставки",
 *                 example="123 Main St"
 *             )
 *         )
 *     ),
 *     @OA\Response(
 *         response=201,
 *         description="Заказ успешно создан",
 *         @OA\JsonContent(
 *             type="object",
 *             @OA\Property(
 *                 property="orderId",
 *                 type="integer",
 *                 description="ID созданного заказа",
 *                 example=1001
 *             ),
 *             @OA\Property(
 *                 property="message",
 *                 type="string",
 *                 description="Сообщение об успешном создании заказа",
 *                 example="Заказ успешно создан"
 *             )
 *         )
 *     ),
 *     @OA\Response(
 *         response=400,
 *         description="Неверные данные",
 *         @OA\JsonContent(
 *             type="object",
 *             @OA\Property(
 *                 property="message",
 *                 type="string",
 *                 description="Сообщение об ошибке",
 *                 example="Ошибка валидации данных"
 *             )
 *         )
 *     ),
 *     @OA\Response(
 *         response=404,
 *         description="Пользователь не найден",
 *         @OA\JsonContent(
 *             type="object",
 *             @OA\Property(
 *                 property="message",
 *                 type="string",
 *                 description="Сообщение об ошибке",
 *                 example="Пользователь не найден"
 *             )
 *         )
 *     )
 * )
 */

3. Объяснение основных элементов

  • @OA\Post: Аннотация для POST запроса с указанием пути /api/orders, краткого описания и тегов.
  • @OA\RequestBody: Описание тела запроса, требуемых данных и их структуры. Используется @OA\JsonContent для описания JSON-структуры.
  • userId: Поле с типом integer, представляющее ID пользователя.
  • items: Массив товаров, где каждый товар содержит productId и quantity.
  • shippingAddress: Строка, представляющая адрес доставки.
  • @OA\Response: Описание возможных ответов API с кодами 201 (успех), 400 (ошибка валидации) и 404 (пользователь не найден). В каждом ответе описывается структура возвращаемых данных.

4. Важные моменты при написании документации

Следуя этим рекомендациям, вы сможете правильно оформить Swagger-документацию для вашего API:

  • Всегда указывайте обязательные поля с помощью параметра required.
  • Описывайте каждое поле: тип данных, описание и пример значения.
  • Указывайте возможные коды ответов API и структуру данных для каждого ответа.
  • Группируйте эндпоинты с помощью тегов (например, "Заказы", "Пользователи").

Заключение

Правильное написание Swagger-документации помогает сделать ваше API более понятным для разработчиков и пользователей. Используйте l5-swagger для автоматической генерации документации на основе ваших аннотаций в контроллерах Laravel, следуя стандартам, описанным выше.
